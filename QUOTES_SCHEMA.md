# Quotes payload + schema (Frontend → Backend)

This doc describes the **recommended Quote schema** and **request payloads** generated by the frontend quote builder.

## Core flags (requested)

- **draft**: boolean  
  - `true` when the user clicks **Save as draft**
  - drafts can be edited freely
- **confirmed**: boolean  
  - `true` when the quote is confirmed (either on first create or via later **PATCH/PUT**)

Recommended invariant:

- When `draft === true` → `confirmed` should be `false`
- When `confirmed === true` → `draft` should be `false`

**Frontend behavior:**

- When user clicks **Save as draft** in the quote modal:
  - payload sent with `"draft": true`, `"confirmed": false`
- When user clicks **Confirm book** in the quote modal:
  - payload sent with `"draft": false`, `"confirmed": true`

## Suggested Quote schema (Mongo/Mongoose style)

### Top-level

- `_id`: ObjectId
- `leadId`: ObjectId (ref `Lead`) **required**
- `venueId`: ObjectId (ref `Venue`) **required**
- `createdBy`: ObjectId (ref `User`) (from JWT)

- `bookingType`: `"venue_buyout" | "space_buyout"` **required**
- `spaceId`: ObjectId (ref `Space`) **required if bookingType = "space_buyout"**

- `eventWindow` **required**
  - `startAt`: Date (ISO 8601)
  - `endAt`: Date (ISO 8601)
  - `durationHours`: number (12/24/36/48)

- `pricing` **required**
  - `basePrice`: number
  - `inclusions`: array of `{ name: string, quantity: number, maxQuantity?: number }`
  - `addons`: array of `{ name: string, quantity: number, unitPrice: number }`
  - `gstRate`: number (example: `0.18`)
  - `discount`: number
  - `totals`:
    - `venueBase`: number
    - `venueGst`: number
    - `addonTotal`: number
    - `addonGst`: number
    - `subtotal`: number
    - `discount`: number
    - `total`: number

- **Flags**
  - `draft`: boolean (default `true`)
  - `confirmed`: boolean (default `false`)

- `status`: `"draft" | "shared" | "accepted" | "rejected"` (optional; can be derived from flags too)
- `createdAt`, `updatedAt`: timestamps

## Update Quote payload (Confirm)

When user edits a draft and confirms it, backend can accept either:

- `PATCH /quotes/{quoteId}` (partial update), or
- `PUT /quotes/{quoteId}` (full replace)

### Minimal confirm (PATCH)

```json
{
  "confirmed": true,
  "draft": false
}
```
